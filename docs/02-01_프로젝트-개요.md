## 📋 프로젝트 정보

- **프로젝트명**: 온라인 쇼핑몰 웹사이트 제작
- **난이도**: ⭐⭐⭐⭐ (상)
- **예상 소요 시간**: 20-24시간
- **주요 기술**: HTML, CSS, JavaScript, 반응형 디자인

---

## 🎯 프로젝트 목표

이 프로젝트를 완료하면 다음을 할 수 있습니다:

- 복잡한 웹사이트 구조 설계 및 구현(페이지/컴포넌트/데이터 흐름)
- 상품 목록/상세/장바구니 등 핵심 플로우 구현
- 필터/정렬/페이지네이션 등 사용자 인터랙션 처리
- 반응형 쇼핑몰 UI 제작 및 QA
- 실무에서 중요한 “예외 처리/상태 관리/성능” 감각 익히기

---

## 📖 프로젝트 배경

온라인 쇼핑몰은 전자상거래의 핵심 서비스입니다.  
이 프로젝트에서는 “이커머스에서 자주 등장하는 화면과 로직”을 직접 구현하면서,  
단순 페이지 제작을 넘어서 **데이터 기반 UI + 상태 변화(장바구니) + UX 흐름**을 경험합니다.

## ✅ 완료 기준 (Definition of Done) ← (추가: 상 난이도에 필수)

아래 항목을 **모두** 만족하면 “프로젝트 완료”로 봅니다.

### 기능 완료

- [ ] 메인 → 목록 → 상세 → 장바구니까지 핵심 플로우가 끊김 없이 동작한다.
- [ ] 필터/정렬/페이지네이션이 조합되어도 올바른 결과가 나온다.
- [ ] 장바구니 합계(총액/수량)가 항상 정확히 계산된다.
- [ ] 새로고침해도 장바구니가 유지된다(localStorage 등).

### UX/예외 처리

- [ ] 로딩/빈 상태/에러 상태 UI가 있다(예: “상품이 없습니다”, “불러오기 실패”).
- [ ] 클릭/터치 영역이 충분히 크다(모바일에서 사용 가능).
- [ ] 이미지 로딩 실패 시 대체 이미지(placeholder)가 표시된다.

### 품질(최소 기준)

- [ ] Lighthouse(모바일) 기준: Performance 70+ / Accessibility 85+ / SEO 80+
- [ ] 키보드로 주요 기능 이용 가능(포커스 스타일 포함)
- [ ] 외부 링크에 `rel="noopener noreferrer"` 적용
- [ ] 콘솔 에러 0개(경고 일부는 허용 가능)

## 📝 요구사항 정의

### 0) 공통 요구사항 (추가)

- **데이터 단위 정의**
  - 상품: `id, name, price, image, category, description, rating(선택), stock(선택)`
- **가격 표기 규칙**
  - 원화 표기(예: `₩12,000`) + 천 단위 콤마
- **상태(상황) 처리**
  - 로딩 중 / 결과 없음 / 오류 / 성공(토스트) 상태 최소 1개 이상 제공

---

### 필수 기능

1. **메인 페이지**
   - 헤더 (로고, 네비게이션, 검색, 장바구니 아이콘)
   - 히어로 배너
   - 인기 상품 섹션(예: 상위 8개)
   - 카테고리 섹션(카테고리 진입 버튼)
   - 푸터(회사정보/문의/정책 링크는 더미 가능)

2. **상품 목록 페이지**
   - 상품 그리드 레이아웃
   - 필터링 (카테고리, 가격 범위)
   - 정렬 (인기순/최신순/가격 낮은순/높은순)
   - 페이지네이션 또는 더보기(둘 중 1개는 반드시)

   ✅ 권장 UX(추가)
   - 필터 초기화 버튼
   - 검색 결과 “n개 상품” 표시
   - 스켈레톤(또는 로딩 스피너)

3. **상품 상세 페이지**
   - 상품 이미지(대표 1장 + 썸네일 선택은 선택)
   - 상품 정보 (이름, 가격, 설명)
   - 수량 선택(+/-)
   - 장바구니 추가 버튼(성공 피드백)
   - 관련 상품 추천(같은 카테고리 4개 등)

   ✅ 필수 예외(추가)
   - 존재하지 않는 id 접근 시: “상품을 찾을 수 없음” 안내

4. **장바구니 페이지**
   - 장바구니 아이템 목록
   - 수량 조절
   - 아이템 삭제(1개/전체 비우기 중 1개는 필수)
   - 총 금액 계산(상품 합계)
   - 주문하기 버튼(결제는 심화에서 구현, 여기선 이동/알림 처리)

   ✅ 장바구니 저장(추가)
   - localStorage에 장바구니 저장/복원

5. **반응형 디자인**
   - 모바일 (375px 이상)
   - 태블릿 (768px 이상)
   - 데스크톱 (1024px 이상)

## 🎨 디자인 요구사항

### 컬러 시스템

- **Primary**: 브랜드 컬러 (예: #0066FF)
- **Secondary**: 보조 컬러 (예: #FF6600)
- **Neutral**: 그레이 스케일 (#000000 ~ #FFFFFF)
- **Semantic**: Success (#00CC66), Warning (#FF9900), Error (#FF3333)

✅ 보완 포인트(추가)

- “Semantic 컬러”는 토스트/에러 메시지/상태 배지에 실제로 사용해보기
- 다크 모드는 심화로 두되, 컬러는 CSS 변수로 관리 권장

### 타이포그래피

- **Heading**: Bold, 24px~48px
- **Body**: Regular, 16px
- **Caption**: Regular, 14px

✅ 보완 포인트(추가)

- 모바일에서 H1 48px는 클 수 있으니, 반응형 폰트 크기 조절 필요

### 컴포넌트

- 버튼 (Primary, Secondary, Ghost)
- 카드 (상품 카드)
- 입력 필드(검색/필터)
- 모달 또는 토스트(장바구니 추가 확인 등)

✅ 보완 포인트(추가)

- “버튼 상태”: default/hover/disabled/focus-visible 최소 3개는 구현
