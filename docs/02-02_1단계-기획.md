## 1. 사용자 페르소나 정의

### 1.1 페르소나란?

**페르소나(Persona)**는 타겟 사용자를 대표하는 가상의 인물입니다.  
사용자의 니즈, 목표, 행동 패턴을 이해하는 데 도움이 됩니다.

✅ 보완 포인트(중요)

- 페르소나는 “프로필”이 아니라 **의사결정 기준**입니다.
- 따라서 아래 3가지를 꼭 추가하세요:
  1. _무엇이 불편한가?_ (Pain Point)
  2. _무엇이 성공인가?_ (Success Criteria)
  3. _주로 어떤 환경인가?_ (Device/상황)

### 1.2 페르소나 작성 실습

#### 실습 1-1: 주요 페르소나 정의

**페르소나 1: 김민수 (30대 직장인)**

```
이름: 김민수
나이: 32세
직업: IT 회사 개발자
목표: 빠르고 편리하게 필요한 상품 구매
행동 패턴:
- 모바일로 주로 쇼핑
- 가격 비교 중요시
- 빠른 배송 선호
- 간단한 결제 프로세스 선호

(추가) Pain Point:
- 필터가 복잡하면 이탈
- 장바구니에서 수량 변경이 불편하면 스트레스

(추가) Success Criteria:
- 원하는 상품을 1~2분 안에 찾고 장바구니에 담을 수 있다
- 가격/배송 정보가 명확하다
```

**페르소나 2: 이영희 (20대 대학생)**

```
이름: 이영희
나이: 24세
직업: 대학생
목표: 합리적인 가격으로 쇼핑
행동 패턴:
- 할인 정보에 민감
- 리뷰와 평점 중요시
- SNS 공유 선호
- 다양한 상품 탐색

(추가) Pain Point:
- 가격 정렬/할인 표시가 불명확하면 신뢰가 떨어짐
- 리뷰가 없으면 구매 결정을 못함(대체 정보 필요)

(추가) Success Criteria:
- 정렬/필터로 마음에 드는 상품을 빠르게 좁힐 수 있다
- 카드에서 핵심 정보(가격/평점/태그)가 한눈에 보인다
```

**체크리스트:**

- [ ] 주요 페르소나 2-3개 정의 완료
- [ ] 각 페르소나의 목표/불편(Pain)/성공 기준(Success) 정리 완료
- [ ] 각 페르소나가 주로 쓰는 기기(모바일/PC) 명시 완료

## 2. 기능 명세서 작성

### 2.1 기능 명세서란?

**기능 명세서**는 웹사이트에 구현할 기능들을 “나열”하는 문서가 아니라,  
**사용자 행동 → 화면 변화 → 데이터 변화**를 구체적으로 적는 문서입니다.

✅ 보완 포인트(상 난이도에서 필수)  
기능마다 최소 아래를 적어주세요.

- 입력(Input): 사용자가 하는 행동/입력값
- 출력(Output): 화면에 보이는 결과
- 상태(State): 로딩/빈 상태/오류/성공
- 예외(Exception): 잘못된 입력/데이터 없음/네트워크 실패 등

## 3. 사이트맵 구성

### 3.1 사이트맵이란?

사이트맵은 웹사이트의 전체 구조를 시각적으로 표현한 것입니다.

### 3.2 사이트맵 작성 실습

#### 실습 3-1: 사이트맵 그리기

사이트맵 구조(현재 구현됨):

```
쇼핑몰 웹사이트
│
├── 메인 페이지 (index.html)
│   ├── 히어로 배너 (Swiper 슬라이더, 4배너, 자동재생)
│   ├── 퀵 카테고리 (전체보기, 추천, 세일, 뷰티, 스포츠, 아울렛, 슈즈)
│   ├── 인기 상품 섹션 (Swiper 슬라이더)
│   ├── 시즌 추천 섹션 (Swiper 슬라이더)
│   ├── 세일 배너
│   └── 매거진 프리뷰
│
├── 상품
│   ├── 전체 상품 (products.html)
│   ├── 카테고리별 (products.html?category=electronics)
│   ├── 태그별 (products.html?tag=sale, ?tag=new, ?tag=seasonal)
│   ├── 검색 결과 (products.html?q=keyword)
│   └── 상품 상세 (product-detail.html?id=101)
│
├── 장바구니 (cart.html)
│
├── 주문
│   ├── 결제 페이지 (checkout.html)
│   ├── 주문 완료 (order-complete.html)
│   └── 주문 내역 (order-history.html)
│
├── 사용자
│   ├── 로그인 (login.html)
│   ├── 회원가입 (signup.html)
│   └── 마이페이지 (mypage.html)
│
├── 콘텐츠
│   └── 매거진 (magazine.html)
│
└── 기타
    └── 준비중 페이지 (coming-soon.html)
```

✅ 보완 포인트(추가)

- 실제 구현이 정적 페이지 기반이라면:
  - 상세 페이지: `product-detail.html?id=101`
  - 검색/필터: `products.html?category=clothing&sort=price_asc`
- 처럼 “쿼리스트링” 중심으로 가는 것이 안정적입니다.

**Notion에서 사이트맵 작성:**

1. Notion 새 페이지 생성: "사이트맵"
2. 다이어그램 또는 불릿 포인트로 구조 작성
3. 각 페이지에 “역할 + 핵심 CTA(다음 행동)” 추가

**체크리스트:**

- [ ] 사이트맵 작성 완료
- [ ] 각 페이지의 역할 + 다음 행동(CTA) 정의 완료

## 4. 유저플로우 정의

### 4.1 유저플로우란?

**유저플로우(User Flow)**는 사용자가 웹사이트에서 목표를 달성하기 위해 거치는 단계별 경로를 시각화한 것입니다.  
사이트맵이 "구조"를 보여준다면, 유저플로우는 "사용자의 행동 경로"를 보여줍니다.

✅ 보완 포인트(중요)

- 유저플로우는 **"시작점 → 목표 달성"**까지의 경로입니다.
- 각 단계에서 사용자가 **무엇을 보고, 무엇을 클릭하며, 어떤 결정을 내리는지** 명확히 해야 합니다.
- **예외 상황**(상품 없음, 장바구니 비어있음, 오류 등)도 포함해야 합니다.

### 4.2 주요 유저플로우 작성 실습

#### 실습 4-1: 핵심 구매 플로우 (Happy Path)

**플로우 1: 메인 페이지에서 상품 구매**

```
1. 메인 페이지 접속 (/)
   └─> 히어로 배너 확인
   └─> 인기 상품 섹션 스크롤

2. 인기 상품 카드 클릭
   └─> 상품 상세 페이지로 이동 (/product-detail.html?id=101)

3. 상품 상세 페이지
   └─> 상품 이미지/정보 확인
   └─> 가격/할인 정보 확인
   └─> 리뷰/평점 확인
   └─> 수량 선택 (기본값: 1)

4. "장바구니에 추가" 버튼 클릭
   └─> 성공 메시지 모달 표시
   └─> 헤더 장바구니 아이콘 개수 업데이트

5. 장바구니 아이콘 클릭
   └─> 장바구니 페이지로 이동 (/cart.html)

6. 장바구니 페이지
   └─> 담은 상품 목록 확인
   └─> 수량 조절 (증가/감소)
   └─> 상품 금액 합계 확인
   └─> 배송비 계산 확인
   └─> 총 주문 금액 확인

7. "주문하기" 버튼 클릭
   └─> 주문 페이지로 이동 (/checkout.html)
   └─> 또는 주문 완료 모달 표시 [기본]
```

**예외 상황:**

- 상품이 품절인 경우: "품절" 표시, "장바구니 추가" 버튼 비활성화
- 장바구니가 비어있는 경우: "장바구니가 비어있습니다" 메시지, "쇼핑 계속하기" 버튼
- 수량이 0 이하인 경우: 수량 1로 고정, 경고 메시지

**체크리스트:**

- [ ] 핵심 구매 플로우 단계별 정의 완료
- [ ] 예외 상황 3개 이상 정의 완료

#### 실습 4-2: 검색 기반 구매 플로우

**플로우 2: 검색으로 상품 찾기**

```
1. 메인 페이지 접속 (/)
   └─> 헤더 검색창 확인

2. 검색어 입력 (예: "후드")
   └─> Enter 키 또는 검색 버튼 클릭

3. 검색 결과 페이지 (/products.html?query=후드)
   └─> 검색 결과 개수 표시
   └─> 필터 영역 확인 (카테고리, 가격 범위)
   └─> 정렬 옵션 확인 (인기순, 가격순 등)

4. 필터 적용 (선택)
   └─> 카테고리 선택: "의류"
   └─> 가격 범위 설정: 20,000원 ~ 50,000원
   └─> URL 업데이트: /products.html?query=후드&category=clothing&min=20000&max=50000

5. 정렬 변경 (선택)
   └─> "가격 낮은순" 선택
   └─> 상품 목록 재정렬

6. 상품 카드 클릭
   └─> 상품 상세 페이지로 이동
   └─> [플로우 1의 3단계부터 이어짐]
```

**예외 상황:**

- 검색 결과가 없는 경우: "검색 결과가 없습니다" 메시지, 추천 상품 표시
- 검색어가 비어있는 경우: 검색 실행 안 함, 경고 메시지
- 필터 적용 후 결과 없음: "조건에 맞는 상품이 없습니다" 메시지, 필터 초기화 버튼

**체크리스트:**

- [ ] 검색 기반 플로우 단계별 정의 완료
- [ ] 필터/정렬 적용 시 URL 변화 정의 완료

---

#### 실습 4-3: 카테고리 탐색 플로우

**플로우 3: 카테고리로 상품 탐색**

```
1. 메인 페이지 접속 (/)
   └─> 카테고리 섹션 확인

2. 카테고리 클릭 (예: "전자제품")
   └─> 카테고리 상품 목록 페이지로 이동 (/products.html?category=electronics)

3. 상품 목록 페이지
   └─> 카테고리명 표시 (예: "전자제품")
   └─> 상품 그리드 확인 (반응형: 모바일 2열, 데스크톱 4열)
   └─> 페이지네이션 확인

4. 상품 카드 호버/탭
   └─> 상품 이미지 확대 효과 (선택)
   └─> 빠른 보기 버튼 표시 (선택)

5. 상품 카드 클릭
   └─> 상품 상세 페이지로 이동
   └─> [플로우 1의 3단계부터 이어짐]

6. 또는 "장바구니에 추가" 버튼 클릭 (카드 내)
   └─> 장바구니에 즉시 추가
   └─> 토스트 메시지 표시
   └─> 현재 페이지 유지 (목록 페이지에 머무름)
```

**예외 상황:**

- 카테고리에 상품이 없는 경우: "이 카테고리에 상품이 없습니다" 메시지
- 페이지네이션 범위 초과: 1페이지로 리다이렉트 또는 404 처리

**체크리스트:**

- [ ] 카테고리 탐색 플로우 단계별 정의 완료
- [ ] 카드 내 장바구니 추가 vs 상세 페이지 이동 구분 완료

---

#### 실습 4-4: 장바구니 관리 플로우

**플로우 4: 장바구니 수정 및 관리**

```
1. 장바구니 페이지 접속 (/cart.html)
   └─> 담은 상품 목록 확인

2. 수량 조절
   └─> "+" 버튼 클릭 → 수량 증가
   └─> "-" 버튼 클릭 → 수량 감소
   └─> 합계 금액 즉시 업데이트
   └─> localStorage 저장

3. 상품 삭제
   └─> "삭제" 버튼 클릭
   └─> 확인 모달 표시 (선택)
   └─> 확인 클릭 → 상품 제거
   └─> 합계 금액 업데이트

4. 장바구니 비우기 (선택)
   └─> "전체 삭제" 버튼 클릭
   └─> 확인 모달 표시
   └─> 확인 클릭 → 모든 상품 제거
   └─> Empty 상태 UI 표시

5. "쇼핑 계속하기" 버튼 클릭
   └─> 메인 페이지 또는 상품 목록 페이지로 이동
```

**예외 상황:**

- 수량이 1인데 "-" 버튼 클릭: 수량 1로 고정, 삭제 안 함 (또는 삭제 모달 표시)
- 장바구니가 비어있는데 "주문하기" 클릭: 버튼 비활성화, 안내 메시지
- localStorage 오류: 경고 메시지, 데이터 복구 시도

**체크리스트:**

- [ ] 장바구니 관리 플로우 단계별 정의 완료
- [ ] 수량/삭제 시 즉시 반영 규칙 정의 완료

---

#### 실습 4-5: 결제 플로우 (Happy Path)

**플로우 5: 장바구니에서 결제까지**

```
1. 장바구니 페이지 접속 (/cart.html)
   └─> 담은 상품 목록 확인
   └─> 총 주문 금액 확인

2. "주문하기" 버튼 클릭
   └─> 로그인 상태 확인
   ├─> (로그인 안 됨) → 로그인 페이지로 이동 (/login.html)
   └─> (로그인 됨) → 결제 페이지로 이동 (/checkout.html)

3. 결제 페이지 (/checkout.html)
   └─> 주문 상품 목록 확인 (수정 불가, 읽기 전용)
   └─> 배송 정보 입력
      - 받는 분 이름 *
      - 연락처 (전화번호) *
      - 배송지 주소 *
      - 상세 주소 *
      - 배송 메모 (선택)
   └─> 결제 방법 선택
      - 신용카드
      - 무통장 입금
      - 계좌이체
   └─> 주문 요약 확인
      - 상품 합계
      - 배송비
      - 총 주문 금액

4. "주문 완료" 버튼 클릭
   └─> 유효성 검사 (이름, 연락처, 주소 필수)
   ├─> (검사 실패) → 오류 메시지 표시, 해당 필드 강조
   └─> (검사 통과) → 주문 처리
      └─> 주문 데이터 저장 (localStorage)
      └─> 장바구니 비우기
      └─> 주문 완료 페이지로 이동 (/order-complete.html 또는 /orders.html)

5. 주문 완료 페이지
   └─> 주문 번호 표시
   └─> 주문 상품 요약
   └─> 배송 예정일 안내
   └─> "주문 내역 보기" 버튼
   └─> "쇼핑 계속하기" 버튼
```

**예외 상황:**

- 로그인 안 됨: "로그인이 필요합니다" 알림, 로그인 페이지로 이동
- 장바구니가 비어있음: "장바구니가 비어있습니다" 메시지, 장바구니 페이지로 리다이렉트
- 필수 정보 누락: 해당 필드 빨간 테두리, 오류 메시지 표시
- 주문 처리 실패: "주문 처리 중 오류가 발생했습니다" 메시지, 재시도 버튼
- 결제 방법 미선택: "결제 방법을 선택해주세요" 메시지

**체크리스트:**

- [ ] 결제 플로우 단계별 정의 완료
- [ ] 로그인 필수 여부 명확히 정의 완료
- [ ] 배송 정보 입력 필드 정의 완료
- [ ] 결제 방법 선택 옵션 정의 완료
- [ ] 예외 상황 5개 이상 정의 완료

#### 실습 4-6: 회원가입 및 로그인 플로우

**플로우 6: 회원가입**

```
1. 메인 페이지 또는 헤더에서 "회원가입" 클릭
   └─> 회원가입 페이지로 이동 (/signup.html)

2. 회원가입 페이지
   └─> 회원가입 폼 확인
   └─> 필수 정보 입력
      - 이름 *
      - 이메일 *
      - 비밀번호 * (최소 4자 이상)
      - 비밀번호 확인 * (비밀번호와 일치해야 함)
   └─> 약관 동의 (선택)
      - 이용약관 동의 ☐
      - 개인정보처리방침 동의 ☐

3. "회원가입" 버튼 클릭
   └─> 유효성 검사
   ├─> (이메일 중복) → "이미 가입된 이메일입니다" 메시지
   ├─> (비밀번호 불일치) → "비밀번호가 일치하지 않습니다" 메시지
   ├─> (최소 길이 미달) → "비밀번호는 최소 4자 이상이어야 합니다" 메시지
   └─> (검사 통과) → 회원가입 처리
      └─> 사용자 데이터 저장 (localStorage)
      └─> "회원가입 완료! 로그인 페이지로 이동합니다" 알림
      └─> 로그인 페이지로 이동 (/login.html)
```

**예외 상황:**

- 이메일 중복: "이미 가입된 이메일입니다" 메시지, 이메일 필드 강조
- 비밀번호 불일치: "비밀번호가 일치하지 않습니다" 메시지, 비밀번호 확인 필드 강조
- 필수 정보 누락: 해당 필드 빨간 테두리, 오류 메시지 표시
- 비밀번호 최소 길이 미달: "비밀번호는 최소 4자 이상이어야 합니다" 메시지

**플로우 7: 로그인**

```
1. 메인 페이지 또는 헤더에서 "로그인" 클릭
   └─> 로그인 페이지로 이동 (/login.html)

2. 로그인 페이지
   └─> 로그인 폼 확인
   └─> 로그인 정보 입력
      - 이메일 *
      - 비밀번호 *

3. "로그인" 버튼 클릭
   └─> 유효성 검사
   ├─> (이메일/비밀번호 불일치) → "이메일 또는 비밀번호가 올바르지 않습니다" 메시지
   └─> (검사 통과) → 로그인 처리
      └─> 세션 저장 (sessionStorage)
      └─> "로그인 성공!" 알림
      └─> 메인 페이지로 이동 (/) 또는 이전 페이지로 복귀

4. 로그인 후
   └─> 헤더에 사용자 이름 표시 (선택)
   └─> "로그아웃" 버튼 표시
   └─> 장바구니/주문 내역 접근 가능
```

**예외 상황:**

- 이메일/비밀번호 불일치: "이메일 또는 비밀번호가 올바르지 않습니다" 메시지
- 필수 정보 누락: 해당 필드 빨간 테두리, 오류 메시지 표시
- 로그인 실패: 재시도 안내 메시지, "비밀번호 찾기" 링크 (선택)

**플로우 8: 로그아웃**

```
1. 헤더에서 "로그아웃" 버튼 클릭
   └─> 확인 모달 표시 (선택)
   ├─> (취소) → 모달 닫기
   └─> (확인) → 로그아웃 처리
      └─> 세션 데이터 삭제 (sessionStorage)
      └─> "로그아웃 되었습니다" 알림
      └─> 메인 페이지로 이동 또는 페이지 새로고침
```

**체크리스트:**

- [ ] 회원가입 플로우 단계별 정의 완료
- [ ] 로그인 플로우 단계별 정의 완료
- [ ] 로그아웃 플로우 단계별 정의 완료
- [ ] 각 플로우의 예외 상황 포함 완료
- [ ] 유효성 검사 규칙 명확히 정의 완료

### 4.3 유저플로우 다이어그램 작성

#### 실습 4-5: Notion에서 유저플로우 시각화

**Notion에서 작성하는 방법:**

1. **플로우차트 블록 사용**

   - `/flowchart` 입력
   - 각 단계를 박스로 표현
   - 화살표로 연결

2. **텍스트 기반 다이어그램 (간단)**

```
[메인 페이지]
    ↓ (인기 상품 클릭)
[상품 상세 페이지]
    ↓ (장바구니 추가)
[장바구니 페이지]
    ↓ (주문하기)
[주문 완료]
```

## 💡 문제 해결 가이드

### Q1. Notion 데이터베이스가 복잡해졌어요

A: 페이지를 여러 하위 페이지로 분리하거나, 뷰(View)를 활용하여 필터링하세요.  
추가로 “이번 주 할 일” 뷰를 만들면 우선순위가 정리됩니다.

### Q2. GitHub에 push가 안 돼요

A:

- 인증 확인 (Personal Access Token 필요할 수 있음)
- 원격 저장소 URL 확인: `git remote -v`
- 최신 변경사항 먼저 pull: `git pull origin main`

### Q3. Slack 알림이 너무 많아요

A: PR/이슈만 알림 받도록 줄이거나, 이벤트를 최소화하세요.
