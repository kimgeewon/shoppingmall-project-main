# 2단계: 디자인 - 쇼핑몰 프로젝트

## 📚 학습 목표

이 단계를 완료하면 다음을 할 수 있습니다:

- Figma로 와이어프레임 제작(정보 구조/우선순위/구매 플로우 중심)
- 디자인 시스템 구축(색/폰트/간격 + 상태/토큰/접근성까지)
- 프로토타입 제작(핵심 사용자 흐름 검증)
- 디자인 리뷰 및 피드백 반영(근거 기반 개선)

---

## 0. 이 단계에서 반드시 만들어야 하는 “산출물”

쇼핑몰(⭐⭐⭐⭐)은 화면이 많고 상태가 복잡해서, 디자인 단계에서 아래 산출물이 있어야 개발이 흔들리지 않습니다.

### 필수 산출물

- [ ] 와이어프레임: 메인 / 목록 / 상세 / 장바구니 (최소 4개)
- [ ] 디자인 시스템: 컬러/타이포/간격/그리드 + 컴포넌트 상태(hover/disabled/error)
- [ ] 핵심 컴포넌트: 버튼/입력/상품카드/필터UI/배지/모달/페이지네이션
- [ ] 프로토타입: “상품 탐색→상세→장바구니 담기→장바구니 수정” 흐름
- [ ] 반응형: Desktop(1440), Tablet(768), Mobile(375)
- [ ] 디자인 QA 체크: 대비/터치타겟/레이아웃 깨짐 기준

## 1. 와이어프레임 제작

### 1.1 메인 페이지 와이어프레임

#### 실습 1-1: 메인 페이지 구조 설계

**단계별 가이드(원본 유지 + 보완):**

1. **Figma 새 파일 생성**
   - 파일명: "쇼핑몰 - 와이어프레임"
   - 데스크톱 프레임 생성 (1440x1024)

2. **레이아웃 구조**

```
┌─────────────────────────────┐
│ 헤더 (Header)               │
│ - 로고, 네비, 검색, 장바구니 │
├─────────────────────────────┤
│ 히어로 배너 (Hero Banner)    │
├─────────────────────────────┤
│ 인기 상품 섹션               │
│ [상품] [상품] [상품] [상품]  │
├─────────────────────────────┤
│ 카테고리 섹션                │
│ [카테고리] [카테고리] ...    │
├─────────────────────────────┤
│ 푸터 (Footer)               │
└─────────────────────────────┘
```

3. **기본 도형으로 구조 표현**
   - 사각형으로 각 섹션 구분
   - 텍스트로 레이블 추가
   - 회색 톤 사용

✅ 보완(중요): 메인에서 “구매로 이어지는 정보 우선순위”

- 헤더: 로고/검색/장바구니는 **항상 보이는 핵심**
- 히어로 배너는 예쁘기보다 “카테고리/이벤트 진입” 역할이 명확해야 함
- 인기 상품 카드에는 최소 정보(이미지/이름/가격/평점/배지)가 필요

✅ 와이어프레임에 반드시 표기할 상태

- 로딩 상태: 카드가 회색 스켈레톤으로 보이는지
- 품절 상태: “품절 배지 + 버튼 비활성” 같은 표현
- 할인 상태: 원가/할인가 표현 자리 확보

**체크리스트:**

- [ ] 메인 페이지 와이어프레임 완료
- [ ] 로딩/품절/할인 같은 “상태 자리”까지 확보 완료

#### 실습 1-2: 상품 목록 페이지 와이어프레임

**레이아웃 구조(원본 유지 + 보완):**

- 왼쪽: 필터 사이드바
- 오른쪽: 상품 그리드 (3-4열)

✅ 보완(필수): 목록 페이지는 “탐색 효율”이 핵심  
와이어프레임에서 아래 영역을 반드시 구분해 주세요.

1) 상단 컨트롤 바(우측 영역 상단)

- 결과 개수(예: 120개)
- 정렬 드롭다운(인기순/가격순/최신순)
- (모바일) 필터 버튼(모달 오픈)

2) 필터 영역(좌측)

- 카테고리(체크/라디오)
- 가격 범위(슬라이더 or 입력)
- 브랜드(체크)
- “필터 초기화” 버튼(강추)
- 적용 중인 필터 칩(예: Clothing / 10,000~50,000) (강추)

3) 상품 그리드

- 카드 동일 높이(텍스트 길이 편차 대응 자리 필요)
- “결과 없음” 상태 UI 영역 확보

4) 페이지네이션(하단)

- 1 2 3 … Next
- 모바일에서는 “더보기”로 대체 가능(선택)

@media (min-width: 1536px) { .container { max-width: 1536px; } }  /* 2xl */

- 거터: 24px

**Mobile (375px ~ 639px)**

- 컬럼: 4

- 좌우 패딩: 16px

## 6. 프로토타입 제작

### 6.1 인터랙션 추가

#### 실습 6-1: 기본 프로토타입 연결

**연결할 인터랙션(원본 유지):**

1. **메인 페이지**
   - 상품 카드 클릭 → 상품 상세 페이지
   - 카테고리 클릭 → 상품 목록 페이지
   - 네비게이션 메뉴 클릭 → 해당 페이지

2. **상품 목록 페이지**
   - 필터 적용 → 상품 목록 업데이트
   - 정렬 변경 → 상품 목록 재정렬

3. **상품 상세 페이지**
   - 장바구니 추가 버튼 → 모달 표시

✅ 보완(핵심 플로우 검증)  
프로토타입의 목표는 “예쁘게 움직이기”가 아니라,  
**사용자가 길을 잃지 않고 구매 흐름을 완주하는지** 확인입니다.

최소 테스트 시나리오(프로토타입에 맞춰 화면 연결)

- 시나리오 A: 메인 → 카테고리 → 목록 필터 → 상세 → 장바구니 담기
- 시나리오 B: 목록에서 정렬 변경 → 상세 진입 → 뒤로가기 → 목록 상태 유지(선택)

**프로토타입 설정(원본 유지):**

- Trigger: On Click
- Action: Navigate to
- Animation: Smart Animate

**체크리스트:**

- [ ] 프로토타입 연결 완료
- [ ] 시나리오 A 완주 가능
- [ ] 모달 닫기/토스트 확인 등 피드백 흐름 확인 완료

## 💡 문제 해결 가이드

### Q1. 컴포넌트 수정이 반영되지 않아요

**A**: 마스터 컴포넌트를 수정해야 합니다. 인스턴스가 아닌 원본 컴포넌트를 찾아 수정하세요.

### Q2. 프로토타입이 작동하지 않아요

**A**: 연결선이 올바르게 연결되었는지, 프레임 이름이 올바른지 확인하세요.

### Q3. 모바일에서 레이아웃이 자꾸 깨져요(추가)

**A**:

- Layout Grid(4컬럼)과 마진(16)을 먼저 고정한 뒤 요소를 배치하세요.
- 버튼/카드의 최소 높이(예: 44px)를 규칙으로 정하면 흔들림이 줄어듭니다.
