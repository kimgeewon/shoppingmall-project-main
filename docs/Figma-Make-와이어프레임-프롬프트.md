# Figma Make 와이어프레임 프로토타입 프롬프트 - 쇼핑몰 프로젝트

## 프로젝트 개요

온라인 쇼핑몰 웹사이트의 UX 와이어프레임 프로토타입을 제작합니다. 사용자가 상품을 탐색하고, 상세 정보를 확인하며, 장바구니에 담아 구매까지 이어지는 핵심 플로우를 중심으로 설계합니다.

---

## 1. 프로젝트 컨텍스트

### 타겟 사용자 (페르소나)
- **주요 사용자 1**: 30대 직장인 (모바일 중심, 빠른 구매 선호, 필터 복잡도에 민감)
- **주요 사용자 2**: 20대 대학생 (할인 정보 민감, 리뷰/평점 중요, 가격 정렬 필수)

### 핵심 목표
- 상품을 빠르게 찾고 장바구니에 담을 수 있어야 함
- 가격/배송 정보가 명확해야 함
- 필터/정렬로 원하는 상품을 쉽게 좁힐 수 있어야 함

### 디바이스
- 데스크톱 (1440px 기준)
- 태블릿 (768px 기준)
- 모바일 (375px 기준)

## 2. 사이트맵 구조 (현재 구현됨)

```
메인 페이지 (index.html)
├── 히어로 배너 (Swiper 슬라이더, 4배너, 자동재생 5초)
├── 퀵 카테고리 (전체보기, 추천, 세일, 뷰티, 스포츠, 아울렛, 슈즈)
├── 인기 상품 섹션 (Swiper 슬라이더)
├── 시즌 추천 섹션 (Swiper 슬라이더)
├── 세일 배너
└── 매거진 프리뷰

상품 목록 (products.html)
├── 전체 상품
├── 카테고리별 (?category=electronics, clothing, food)
├── 태그별 (?tag=sale, ?tag=new, ?tag=seasonal)
└── 검색 결과 (?q=검색어)

상품 상세 (product-detail.html?id=...)
├── 이미지 갤러리 (썸네일 선택)
├── 상품 정보 (이름, 가격, 할인, 평점)
├── 수량 선택
├── CTA 버튼 (장바구니 담기, 바로 구매)
└── 관련 상품

장바구니 (cart.html)
├── 아이템 목록 (수량 변경, 삭제)
└── 주문 요약 (배송비 정책 포함)

주문 플로우
├── 결제 페이지 (checkout.html)
├── 주문 완료 (order-complete.html)
└── 주문 내역 (order-history.html)

사용자
├── 로그인 (login.html)
├── 회원가입 (signup.html)
└── 마이페이지 (mypage.html)

콘텐츠
├── 매거진 (magazine.html)
└── 준비중 (coming-soon.html)
```

**상품 카드 구성 요소:**
- 상품 이미지 (16:9 비율)
- 상품명 (최대 2줄, 말줄임표)
- 가격 (원가 취소선 + 할인가 강조, 할인율 배지)
- 평점 (별점 + 리뷰 수)
- "장바구니 추가" 버튼 (호버 시 표시)
- 상태 배지 (NEW, HOT, SALE, 품절)

**상태 표현:**
- 로딩 상태: 회색 스켈레톤 UI
- 품절 상태: 이미지 오버레이 + "품절" 배지, 버튼 비활성
- 할인 상태: 원가 취소선, 할인가 강조, 할인율 배지

**필터 사이드바 구성:**
- 카테고리 필터 (체크박스 또는 라디오)
- 가격 범위 필터 (슬라이더 또는 최소/최대 입력)
- 브랜드 필터 (체크박스)
- "필터 초기화" 버튼
- 적용 중인 필터 칩 (태그 형태, X 버튼으로 제거)

**상단 컨트롤 바:**
- 결과 개수 표시 ("120개 상품")
- 정렬 드롭다운 (인기순, 가격 낮은순, 가격 높은순, 최신순)

**상품 그리드:**
- 데스크톱: 4열
- 태블릿: 3열
- 모바일: 2열
- 카드 높이 통일 (텍스트 길이 편차 대응)

**상태 표현:**
- 로딩 상태: 스켈레톤 카드 그리드
- 결과 없음: "조건에 맞는 상품이 없습니다" 메시지 + "필터 초기화" 버튼
- 필터 적용 중: 적용된 필터 칩 표시

**모바일 대응:**
- 필터 사이드바 → 모달로 전환
- "필터" 버튼 클릭 시 모달 열림
- 상단 컨트롤 바: 결과 개수 + 정렬 + 필터 버튼

1. 상품명 (H1, 큰 폰트)
2. 평점/리뷰 수 (선택, 별점 + 숫자)
3. 가격 (원가 취소선 + 할인가 강조 + 할인율 배지)
4. 배송 정보 (무료배송 조건, 배송비)
5. 수량 선택 (스텝퍼: - 버튼, 숫자, + 버튼)
6. CTA 버튼 (장바구니 담기, 바로 구매 - 2개 버튼 나란히)
7. 보조 정보 (교환/환불, 상품 설명, 사이즈/재질)

**상태 표현:**
- 품절: CTA 버튼 비활성화, "품절" 배지, "재입고 알림" 버튼 (선택)
- 장바구니 담기 성공: 토스트 메시지 (우측 상단) 또는 모달
- 이미지 로딩 실패: placeholder 이미지 (회색 배경 + 아이콘)

**이미지 갤러리:**
- 메인 이미지 (큰 크기)
- 썸네일 이미지 (작은 크기, 클릭 시 메인 이미지 교체)
- 이미지 확대 기능 (선택)

### 3.4 장바구니 페이지 (cart.html)

- 가격 × 수량
- 수량 스텝퍼 (- 버튼, 숫자, + 버튼)
- 삭제 버튼 (X 아이콘)

**주문 요약 구성:**
- 상품 합계
- 배송비 (정책 문구 포함: "50,000원 이상 무료배송")
- 구분선
- 총 주문 금액 (큰 폰트, 강조)
- "주문하기" 버튼 (Primary, 큰 크기)
- "쇼핑 계속하기" 버튼 (Secondary)

**상태 표현:**
- 빈 장바구니: "장바구니가 비어있습니다" 메시지 + "쇼핑하러 가기" 버튼
- 수량 변경: 즉시 합계 반영 (애니메이션)
- 주문하기 버튼: 장바구니 비어있으면 비활성화

**모바일 대응:**
- 아이템 리스트 (상단)
- 주문 요약 (하단 고정 또는 스크롤 가능)

## 7. 프로토타입 연결 (Figma Make)

### 핵심 플로우 연결:
1. **메인 → 상품 상세**: 인기 상품 카드 클릭 → 상품 상세 페이지
2. **상품 상세 → 장바구니**: "장바구니 담기" 버튼 클릭 → 토스트 표시 → 장바구니 아이콘 업데이트
3. **장바구니 → 결제**: "주문하기" 버튼 클릭 → 로그인 확인 → 결제 페이지
4. **결제 → 주문 완료**: "주문 완료" 버튼 클릭 → 주문 완료 페이지
5. **회원가입 → 로그인**: 회원가입 완료 → 로그인 페이지로 이동
6. **로그인 → 메인**: 로그인 성공 → 메인 페이지로 이동
7. **상품 목록 → 필터**: 필터 적용 → 상품 그리드 업데이트
8. **검색**: 검색창 입력 → 검색 결과 페이지
