# 필요 개념 정리

UI/UX 실습을 진행하기 전에 알아야 할 핵심 개념을 정리합니다.  
(쇼핑몰 맥락으로 요약합니다.)

---

## 1. UI vs UX

- **UI**: 사용자가 보는 화면·요소 (버튼, 레이아웃, 색상)
- **UX**: 사용자의 전체 경험 (쇼핑 흐름, 결제 난이도, 피드백)
- 좋은 UI는 좋은 UX의 일부입니다.

---

## 2. 디자인 시스템·디자인 토큰

- **디자인 토큰**: 색상·간격·타이포 등을 변수로 정의 (`variables.css`의 `--color-primary`, `--spacing-md` 등)
- **장점**: 한 곳만 수정해도 전체 UI에 반영 (쇼핑몰 브랜드 색 통일)

---

## 3. 반응형 디자인

- **브레이크포인트**: 모바일(~767px), 태블릿(768~1023px), 데스크톱(1024px~)
- 상품 그리드·헤더·장바구니 UI가 화면 크기에 맞게 조정되도록 합니다.

---

## 4. 접근성

- 이미지 `alt`, 키보드 포커스, 터치 타겟 크기(최소 44px), 시맨틱 HTML (`header`, `nav`, `main`, `footer`)

---

## 5. 유저플로우 (쇼핑몰)

### 예: "상품 구매" 플로우

```
메인/상품 목록 → 상품 상세 → 장바구니 담기 → 장바구니 → 결제 → 주문 완료
```

- **단계 수**, **피드백**(장바구니 담기 모달, 주문 완료 메시지), **오류 처리**, **빈 상태**(장바구니 비었을 때)를 고려합니다.

---

## 6. 컴포넌트

- 재사용 가능한 UI 단위: 버튼(.btn), 상품 카드(.product-card), 폼, 모달
- 상태: Default, Hover, Focus, Disabled, Loading

---

## 7. 시맨틱 HTML

- `<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`, `<button>`, `<a>` 등 의미에 맞게 사용

---

## 8. CSS 변수·Flexbox·Grid

- **CSS 변수**: `:root { --color-primary: #000; }` → `var(--color-primary)`
- **Flexbox**: 1차원 정렬 (헤더, 버튼 행)
- **Grid**: 2차원 레이아웃 (상품 그리드)
