# 10단계: 최종 완성 및 테스트

전체 기능을 점검하고, 접근성·반응형·로컬 환경을 최종 확인합니다.

---

## 학습 목표

- 로컬 서버에서 전체 플로우 테스트
- 접근성·반응형 최종 확인
- (선택) AppHeader, AppFooter 웹 컴포넌트 적용

---

## 1. 전체 플로우 테스트

### 1-1. 쇼핑 플로우

1. **메인(index.html)**: 히어로, 인기 상품·시즌 추천 슬라이드, 매거진 표시. "장바구니 담기" 클릭 시 모달.
2. **상품 목록(products.html)**: 필터/정렬, 카드 클릭 → product-detail.html?id=1
3. **상품 상세(product-detail.html)**: 상품 정보, 장바구니 담기, "상품 목록" 브레드크럼 → products.html
4. **장바구니(cart.html)**: 목록·수량·삭제·합계, "결제하기" → checkout.html
5. **결제(checkout.html)**: 주문 요약·결제 처리 → order-complete.html

### 1-2. 로그인·마이페이지

1. **로그인(login.html)**: 성공 시 메인 또는 redirect
2. **회원가입(signup.html)**: 가입 후 로그인 가능
3. **마이페이지(mypage.html)**: 로그인 시 접근
4. **주문 내역(order-history.html)**: 주문 목록 표시

### 1-3. 기타

- **매거진(magazine.html)**, **준비 중(coming-soon.html)** 표시 확인

## 2. 기능별 체크리스트

| 구분                | 확인 항목                                              |
| ------------------- | ------------------------------------------------------ |
| 메인                | 히어로, 인기/시즌 슬라이드, 매거진, 장바구니 담기 모달 |
| 상품 목록           | 필터·정렬, 카드 클릭 시 상세 이동                      |
| 상품 상세           | id 파라미터 표시, 상품 목록 링크, 장바구니 담기        |
| 장바구니            | 목록·수량·삭제·합계, 결제하기 이동                     |
| 결제                | 주문 요약·결제 처리·주문 완료 페이지                   |
| 로그인/회원가입     | 유효성 검사, 리다이렉트                                |
| 마이페이지/주문내역 | 로그인 시 데이터 표시                                  |

## 3. 로컬 서버·CORS 확인

- file:// 대신 로컬 서버 사용. ES 모듈·fetch 시 CORS 방지를 위해 필수.
- data/products.json, data/users.json 로드 및 모듈 로드 오류 없음 확인.

---

## 4. 접근성·반응형

- 키보드 포커스, 이미지 alt, 폼 label
- 375px, 768px, 1024px에서 레이아웃 확인
